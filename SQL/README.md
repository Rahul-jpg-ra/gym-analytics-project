# 🗄️ Gym Analytics - SQL Analysis

## 📌 Project Overview

This project demonstrates SQL-based analysis of a Gym Management database. It focuses on solving real-world business problems using SQL queries, joins, subqueries, window functions, CTEs, views, and triggers.

# 📈 Business Questions Solved

## Membership Analysis

- Find the total number of gym members.
- Count Active vs Inactive members.
- Find all active members.
- Find all female members.
- Find active female members.
- Find members who joined after a specific date.
- Categorize members by age using CASE.
- Find the city with the highest number of active members.

## Revenue Analysis

- Calculate total gym revenue.
- Find the membership plan generating the highest revenue.
- Calculate average payment for each membership plan.
- Categorize payments into Premium and Standard.

## Trainer Analysis

- Find the trainer handling the highest number of members.
- Find trainers managing more than 50 members.
- Find the highest-paid trainer.
- Calculate average trainer salary.
- Display every member with their assigned trainer.
- Show trainers without assigned members.

## Fitness Analysis

- Calculate average BMI.
- Find members whose BMI is above average.
- Find the most common fitness goal.
- Rank members based on BMI.
- Rank members by BMI within each city.

## Membership Plan Analysis

- Find members with Premium memberships.
- Find membership plans that have never been purchased.

## Payment Analysis

- Show previous payment using LAG().
- Show next payment using LEAD().
- Show the latest payment made by every member.

## Advanced SQL Analysis

- Top 3 BMI members from each city.
- Highest-paid trainer from each specialization.
- Latest payment of each member using CTE.
- Member ranking using Window Functions.
